  <h3>📊 Task 1: Logistic Regression – High vs Low Spending Prediction</h3>
  <p>
    Objective: Build a binary logistic regression model to classify Fresco Supermarket customers as high or low spenders based on purchasing behavior.
  </p>
  <ul>
    <li><strong>Tools Used:</strong> SPSS</li>
    <li><strong>Key Features:</strong> Value & Brand product counts</li>

  </ul>

  <h3>📈 Task 2: Time Series Decomposition – Airline Passenger Forecast</h3>
  <p>
    Objective: Use decomposition to forecast monthly air passenger demand for 1960 using historical data from 1949–1959.
  </p>
  <ul>
    <li><strong>Trend:</strong> Upward over time</li>
    <li><strong>Seasonality:</strong> 12-month cycle</li>
    <li><strong>Model:</strong> Multiplicative</li>
    <li><strong>Accuracy:</strong> Evaluated via MAE and MSE</li>
  </ul>

  <h3>🧮 Task 3: ARIMA Forecasting – UK COVID-19 Cases</h3>
  <p>
    Objective: Forecast daily COVID-19 cases for 15–21 June 2020 using ARIMA modeling on EU open data.
  </p>
  <ul>
    <li><strong>Model Chosen:</strong> ARIMA(1,1,0)</li>
    <li><strong>R²:</strong> 0.878</li>
    <li><strong>Forecast Example:</strong> 15 June: 1470 cases</li>
    <li><strong>Validation:</strong> Residuals showed no significant autocorrelation (Ljung-Box p = 0.010)</li>
  </ul>

  <h3>🧠 Task 4: Artificial Neural Network – Exchange Rate Forecasting</h3>
  <p>
    Objective: Predict the US/UK exchange rate using a Multilayer Perceptron Neural Network with lagged input features.
  </p>
  <ul>
    <li><strong>Input Variables:</strong> 4 lagged exchange rates</li>
    <li><strong>Network:</strong> 4-3-1 architecture with tanh & identity activations</li>
    <li><strong>Forecast Output:</strong> 1.3114 USD/GBP on 8 August 2020</li>
    <li><strong>Insight:</strong> Lag 1 had the highest normalized importance (100%)</li>
  </ul>

  <h3>👥 Task 5: Cluster Analysis – Bank Customer Segmentation</h3>
  <p>
    Objective: Identify segments in UK bank customers using both Hierarchical and Two-Step clustering.
  </p>
  <ul>
    <li><strong>Variables:</strong> Marital Status, Account Balances, Employment</li>
    <li><strong>Clusters:</strong> 3 (High Net Worth, Youth & Low Income, At-Risk Customers)</li>
    <li><strong>Recommendations:</strong> Personalised products and services for each segment</li>
  </ul>

  <h3>📱 Task 6: Conjoint Analysis – Mobile Phone Product Preferences</h3>
  <p>
    Objective: Identify consumer preferences for mobile phone features using ranking-based linear regression.
  </p>
  <ul>
    <li><strong>Attributes:</strong> Brand, Storage, Battery Life, Price</li>
    <li><strong>Top Drivers:</strong> Battery Life (37.7% importance)</li>
    <li><strong>Most Preferred Features:</strong> 128–256GB Storage, 24–26 hrs Battery, Price £899</li>
    <li><strong>Insights:</strong> Consumers cared more about specs than brand</li>
  </ul>
